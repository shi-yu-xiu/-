 # React + Vite 作品集项目常见问题
 
 ## 1. 环境与依赖
 - npm 被 PowerShell 执行策略阻止 → 用 `& "C:\Program Files\nodejs\npm.cmd"` 绕过
 - 无网络时用 standalone HTML 交付，有网络后迁移回 React+Vite
 
 ## 2. CSS 维护
 - ❌ 不要用 substring 批量替换 CSS 内容 → 容易删除相邻样式（hero/navbar丢失）
 - ✅ 从 `portfolio.html` 提取原始样式 + 增量追加新规则
 - GSAP 残留规则 `.hero-title{opacity:0}` 会导致首屏全透明 → 使用后必须彻底清理
 
 ## 3. React 组件陷阱
 - ❌ Hooks 不能放在条件分支（`if(sub)`）内 → 违反 hooks 调用顺序规则 → 提取为独立子组件
 - ❌ BorderGlow z-index 层级错误：发光层在卡片 background 后面 → 用 `z-index:2` 放在内容层之上
 - ❌ 多次 string.replace 容易导致 JSX 不闭合 → 改用 writeFileSync 全量重写
 - ✅ main.jsx 始终保留 `import React from 'react'`，否则 `<React.StrictMode>` 报错
 
 ## 4. 状态管理与动画
 - ✅ IntersectionObserver + `.reveal`/`.visible` 实现滚动渐入，不要移除观察器
 - ❌ GSAP 集成风险高：版本兼容、hooks 冲突、错误难排查 → 纯 CSS 更稳定
 - ✅ 图片文件名避免中文 → 用 `w01.png`、`sofa_01.jpg`
 
 ## 5. 调试方法
 - 页面空白时先查 main.jsx 编译是否 200，再查 `window.onerror`
 - 用 `fetch('/src/App.jsx').then(r=>r.status)` 验证模块是否正常
 - IntersectionObserver 被删除 → 所有 `.reveal` 元素不可见


## 6. 数据操作安全
- ❌ 永远不要用 `sed -i '/^\s*,/d'` 或 `replace(/^\s*,\n/gm, '')` 清理数据文件
  - 会误删 CarouselView 组件（不以逗号开头但仍被匹配）
  - 会误删子类目数据（平面设计、品牌设计整段丢失）
  - 改用精确字符串替换或全量重写
- ✅ 修改 JSX 数据文件时，用 `fs.writeFileSync` 全量重写更安全
- ✅ 文件扩展名要匹配：`/images/poster_01.png` 不能写成 `poster_01.jpg`

## 7. Vite 服务器稳定性
- Vite dev server 作为子进程运行时，shell 超时会连带杀掉
- 改用 `detached: true + unref()` 启动独立进程
- 或用 `npm run build` + `vite preview` 部署静态站点，更稳定

## 8. 排查白屏/黑屏流程
1. 检查 `main.jsx` 返回 200（语法错误会导致页面完全空白）
2. 检查 `window.addEventListener('error')` 捕获 JS 错误
3. 检查 `.reveal` + IntersectionObserver 是否存在（遗漏会导致内容不可见）
4. 检查关键组件（CarouselView 等）是否被误删
5. 检查数据文件中所有分类/子类目是否完整
6. 检查 build 产物中是否包含预期数据（`dist/assets/index-*.js`）
